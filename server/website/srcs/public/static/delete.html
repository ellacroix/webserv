DELETE method

Here is the description of the DELETE method according to RFC2616:

The DELETE method requests that the origin server delete the resource identified by the Request-URI.

In webserv, a non-cgi DELETE request will remove the resource identified by the request URI from the server's file system.

Those actions are performed only if writing is allowed for the path.

It is then the responsibility of the website administrator to edit the permissions of its file system.

DELETE requests

We could not find a way to test the file DELETEing using HTML forms.

The we will use curl, again to test the DELETE feature.

We will try to remove the previously uploaded file "/path/to/repo/server/test_files/schema.png" at URI "/media/schema.png".

  1. To check that the file still exists, try to GET in Firefox:

    /media/schema.png

    You should see the file you uploaded displayed by the browser.

  2. Now open the terminal and try the following:

    $> curl -v -X DELETE http://localhost:8000/media/schema.png

    This command will try to remove the file schema.png using the DELETE.

    If the request was successful, the response sent to curl will contain a body specifying that the removal was performed for the URI with a "200 OK" status code.

  3. Now try to access the URI from step 1 again.

    You should be returned the default error page for error 404.
